# 可視化・レポートスキル

## 1. 可視化の原則【GL-8】
- グラフの軸は0始まりを原則とする
- 切り詰める場合は「注: 軸は○○から始まっています」と明記
- 色は色覚多様性に配慮したパレットを使用（viridis, colorblind等）

## 2. 不確実性の可視化【GL-4】
- 棒グラフ: 95%信頼区間のエラーバーを必ず付ける
- 折れ線グラフ: 信頼区間の帯（shaded area）を表示
- 散布図: 回帰直線には信頼区間を付ける

## 3. 主要可視化の種類
- 分布確認: ヒストグラム + KDE
- 変数関係: 散布図行列（pair plot）
- モデル評価: ROC曲線・PR曲線・残差プロット
- 特徴量重要度: SHAP ウォーターフォールプロット

## 4. レポート構成
**エグゼクティブサマリー（非技術者向け）:**
- 分析目的・主要な発見・推奨アクション・限界

**技術的詳細レポート（データサイエンティスト向け）:**
- データ前処理の手順・モデルの詳細・評価指標の全数値
