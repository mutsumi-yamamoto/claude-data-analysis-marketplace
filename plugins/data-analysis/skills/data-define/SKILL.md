## 1. Issue の定義【Issue Driven】

「解くべき問い（Central Question）」を一文で定義する:
- 「何を明らかにすれば意思決定できるか」から逆算する
- 「売上を上げたい」→「どの顧客セグメントにどのプロモーションが有効か」のように具体化
- **チェック**: 分析結果でアクションが変わるか？（Actionableか）
- **チェック**: そもそもデータで解ける問題か？

## 2. Issue Tree の構築【MECE】

Central Question を MECE に分解する:

```
Central Question
├── Sub-Issue A: ○○はなぜ起きているのか
│   ├── 仮説A-1: ○○が原因ではないか
│   └── 仮説A-2: ○○が原因ではないか
├── Sub-Issue B: ○○の規模はどのくらいか
└── Sub-Issue C: ○○に対して何ができるか
```

## 3. 仮説リストの設計【Hypothesis Driven】

各Sub-Issueについて:
- **仮説**: 「おそらく○○だろう」
- **根拠**: 「なぜなら○○だから」
- **検証手段**: 「○○データで○○分析を行う」
- **仮説を否定するデータ**: 「○○が見られた場合は仮説が誤り」【GL-7】

|----------|--------|--------|-------------|----------|
| KGI（最終目標） | | | | |
| KPI（主要） | | | | |
| KPI（副次） | | | | |

**成功基準（定量的閾値）**【GL-2, GL-4】:
- 「改善した」ではなく「○○が○○%以上向上」と数値で定義
- 最小検出水準（MDE: Minimum Detectable Effect）を設定する
- 点推定値には95%信頼区間を付ける【GL-4】

## 5. 必要サンプルサイズの試算【GL-5】

```python
from statsmodels.stats.power import TTestIndPower

analysis = TTestIndPower()
n = analysis.solve_power(
    effect_size=0.3,   # MDEをもとに設定
    alpha=0.05,
    power=0.80,        # 検定力 1-β = 0.80
    alternative='two-sided'
)
print(f"必要サンプルサイズ: {n:.0f} / グループ")
```
