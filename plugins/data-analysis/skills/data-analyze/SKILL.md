# フルサイクル分析スキル（Phase 0〜8）

## 前提: analysis_context.md を必ず読み込んでから実行すること【最重要】

コンテキストが不十分な場合はヒアリングを行ってから分析に進む。

---

## 実行フロー

以下のPhaseを順に実行する。各フェーズの完了後、次フェーズに進む前にユーザーに確認を取ること。

---

### Phase 0: 問題定義・目的設定

`/data-analysis:data-define` の内容に従って実施:

1. **Issue の定義**: 「何を明らかにすれば意思決定できるか」を一文で定義
2. **Issue Tree**: MECEに分解、仮説リストを作成【Hypothesis Driven】
3. **KGI/KPI設計**: 定量的成功基準・MDEを設定【GL-2, GL-4】
4. **必要サンプルサイズ試算**【GL-5】
5. **スコープ確定**: 期間・地域・データソース範囲
6. **Actionabilityチェック**: 結果でアクションが変わるか確認

**チェックポイント**: Phase 0 完了 → ユーザー承認 → Phase 1へ

---

### Phase 1: データ理解・概要把握

`/data-analysis:data-explore` の内容に従って実施:

1. **スキーマ確認**: カラム名・データ型・主キー・外部キー
2. **基本統計量**: mean, median, std, min, max, 四分位数
3. **分布の可視化**: ヒストグラム・箱ひげ図【GL-8】
4. **欠損値パターン分類**: MCAR/MAR/MNAR
5. **カーディナリティ確認**
6. **時系列の確認**（該当する場合）
7. **落とし穴チェック**: サバイバーシップバイアス、データ生成過程の理解

**チェックポイント**: Phase 1 完了 → ユーザー承認 → Phase 2へ

---

### Phase 2: データ品質評価・クレンジング

`/data-analysis:data-clean` の内容に従って実施:

1. **品質6次元評価**: 正確性・完全性・一貫性・一意性・妥当性・適時性
2. **クレンジング順序**: 重複削除 → 欠損処理 → 外れ値処理
3. **欠損値処理**: MCAR→リストワイズ可、MAR→MICE推奨、MNAR→モデルベース
4. **外れ値処理**: IQR法・Zスコア法で検出【GL-6】
5. **クレンジング前後の比較**: 分布変化を確認【GL-7】

**チェックポイント**: Phase 2 完了 → ユーザー承認 → Phase 3へ

---

### Phase 3-5: 特徴量設計・エンジニアリング

`/data-analysis:data-feature` の内容に従って実施:

1. **仮説リストに基づく特徴量候補の洗い出し**
2. **ターゲットリーケージの防止**【GL-10】
3. **特徴量エンジニアリングの実施**
   - 数値変換: 対数変換、Box-Cox、ビニング、スケーリング
   - カテゴリエンコーディング
   - 時系列特徴量（ラグ、移動平均、差分）
4. **多重共線性チェック**: VIF > 10 は要注意【GL-9】
5. **Train/Test分割**: スケーリングパラメータはTrainのみから算出

**チェックポイント**: Phase 5 完了 → ユーザー承認 → Phase 6へ

---

### Phase 6: 分析・モデリング

`/data-analysis:data-model` の内容に従って実施:

分析は4段階の深さで実施:

1. **記述的分析**: 何が起きたか（基本統計・クロス集計・コホート分析）
2. **診断的分析**: なぜ起きたか（相関分析・検定・根本原因分析）【GL-1, GL-3, GL-9】
3. **予測的分析**: 何が起きるか（回帰・分類・クラスタリング・時系列）
4. **処方的分析**: 何をすべきか（最適化・シミュレーション）

ベースライン設定 → 候補モデル比較 → 交差検証 → 最良モデル選択

**チェックポイント**: Phase 6 完了 → ユーザー承認 → Phase 7へ

---

### Phase 7-8: 結果解釈・検証

`/data-analysis:data-interpret` の内容に従って実施:

**Phase 7: 結果解釈**
1. **Fact → Insight → Action → Impact** のフレームワークで構造化
2. 禁止: 相関と因果の混同、チェリーピッキング【GL-9, GL-7】
3. 限界・前提条件の明記
4. 反事実の検討

**Phase 8: 検証・バリデーション**
1. 再現性・頑健性・外的妥当性の確認
2. 交差検証の実施（K-Fold / Stratified / Time Series）
3. 常識チェック（ドメイン知識との整合）
